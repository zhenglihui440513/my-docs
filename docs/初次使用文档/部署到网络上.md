下面是一份**纯小白级**教程，把本地已经能跑的 Docsify 文件夹搬到 **GitHub** 并开启 **GitHub Pages**，让所有人都能访问。全程只有 5 步，复制粘贴即可。

------

1. 注册 & 新建仓库（网页操作）
2. 打开 [https://github.com](https://github.com/) 注册/登录
3. 右上角绿色按钮「＋」→ 「New repository」
   - Repository name：随意英文，如 `my-docs`
   - Description：可写“我的文档站”
   - 选 **Public**（公开仓库才能免费开 Pages）
   - **不要勾选** “Initialize this repository with a README”
   - 点「Create repository」
4. 创建后 **不要关闭页面**，第 3 步要用到仓库地址（HTTPS 那一串，形如 https://github.com/你的用户名/my-docs.git）

------

1. 把本地 Docsify 变成 Git 仓库（一次就行） 在 Docsify 根目录（有 index.html 的那一层）空白处
   `右键 → Git Bash Here` 依次输入：

bash

复制

```bash
git init                    # 初始化
git add .                   # 全部文件加入
git commit -m "first docs"  # 提交说明
```

如果这里再出现



复制

```
Author identity unknown
*** Please tell me who you are.
```

就按提示先配一下身份（只用配一次）：

bash

复制

```bash
git config --global user.name "你的英文名"
git config --global user.email "你的邮箱"
```

然后再执行 `git commit -m "first docs"` 即可 。

------

1. 把代码推到 GitHub（复制粘贴 3 行） 继续在小黑框里执行，**把地址换成你自己的**：

bash

复制

```bash
git remote add origin https://github.com/你的用户名/my-docs.git
git branch -M main          # 把默认分支改名 main（GitHub 现在默认叫 main）
git push -u origin main     # 第一次推送，会弹窗让你登录 GitHub 账号
```

看到 `main → main` 完成 100% 就是成功了。

------

1. 开启 GitHub Pages（让网页能访问）
2. 回到刚才的仓库页面 → 顶部菜单「Settings」→ 左侧「Pages」
3. Source 选择 **Deploy from a branch**
   - Branch：选 `main` / `(root)`
   - 点「Save」
4. 页面刷新后，顶部会出现一行蓝底提示：

> Your site is live at [https://你的用户名.github.io/my-docs/](https://xn--6qqv7i14ofosyrb.github.io/my-docs/)

把这条网址发给任何人，他们就能直接访问你的 Docsify 文档了。

------

1. 以后更新文档（3 秒搞定） 文档改完保存后，在 Docsify 目录再开一次 Git Bash：

bash

复制

```bash
git add .
git commit -m "update"
git push
```

GitHub Pages 会自动更新，通常 10 秒左右就能在浏览器里看到新内容，**不用再点任何按钮**。
